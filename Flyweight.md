# Flyweight (Легковес / Приспособленец)
**Категория:** Structural (Структурный)

## 📌 Суть (Intent)
Паттерн **Flyweight** позволяет вместить огромное количество объектов в оперативную память (RAM), разделяя общее состояние (общие данные) между ними, вместо того чтобы хранить эти данные в каждом объекте.

## 🧠 Фундаментальная идея (из Лекции 12)
**Проблема (Пример: Лес деревьев / Текстовый редактор):**
В игре "Лес" нужно нарисовать 1 000 000 деревьев. У каждого дерева есть координаты (X, Y), возраст, а также *Текстура*, *3D-модель* и *Цвет*. Текстура и 3D-модель весят мегабайты. Если каждое из миллиона деревьев будет хранить копию текстуры, оперативная память закончится через секунду.

**Решение:** Разделить состояние объекта на две части:
1. **Intrinsic (Внутреннее состояние):** То, что дублировалось во всех объектах и никогда не меняется (Текстура, Модель, Цвет). Эти данные мы выносим в **один** общий класс (Flyweight).
2. **Extrinsic (Внешнее состояние):** То, что уникально для каждого дерева (Координаты X, Y, Возраст). Эти данные хранятся снаружи (в массиве, векторе) и передаются в методы Flyweight'а как параметры.

## 🛠 Архитектура (Роли)
1. **Flyweight (Легковес):** Хранит `intrinsic` (внутреннее) состояние. Методы этого класса принимают `extrinsic` состояние извне.
2. **FlyweightFactory (Фабрика легковесов):** Хранит пул (словарь/кэш) уже созданных Легковесов. Если клиент просит Легковес с "Зеленой текстурой Дуба", фабрика ищет его в кэше. Если есть — возвращает ссылку. Если нет — создает, сохраняет в кэш и возвращает.
3. **Context / Client (Контекст):** Хранит уникальное `extrinsic` состояние (X, Y) и ссылку на нужный Легковес.

## 💻 Пошаговая реализация и Примеры

### Пример 1: Лес деревьев (Туториал 12)

    // 1. FLYWEIGHT (Внутреннее состояние)
    class TreeType {
        string name;
        string color; // "Тяжелые" данные
    public:
        TreeType(string n, string c) : name(n), color(c) {}
        
        // Метод принимает внешнее состояние (X, Y) как параметры!
        void draw(int x, int y) {
            cout << "Drawing " << color << " " << name << " at " << x << "," << y << "\n";
        }
    };

    // 2. FLYWEIGHT FACTORY (Кэш)
    class TreeFactory {
        static unordered_map<string, TreeType*> treeTypes;
    public:
        static TreeType* getTreeType(string name, string color) {
            string key = name + "_" + color;
            if (treeTypes.find(key) == treeTypes.end()) {
                treeTypes[key] = new TreeType(name, color); // Создаем только если нет
            }
            return treeTypes[key];
        }
    };

    // 3. CONTEXT (Внешнее состояние)
    class Tree {
        int x, y; // Легкие данные
        TreeType* type; // Ссылка на общий тяжелый объект
    public:
        Tree(int x, int y, TreeType* t) : x(x), y(y), type(t) {}
        void draw() { type->draw(x, y); }
    };

В этом примере миллион объектов `Tree` будут занимать минимум места, так как все они будут ссылаться всего на 2-3 объекта `TreeType`, хранящихся в Фабрике.

## ⚖️ Плюсы и Минусы
**Плюсы:**
* Экономит гигантские объемы оперативной памяти, если программа работает с кучей похожих объектов.

**Минусы:**
* Расходует процессорное время (CPU cycles) на поиск/вычисление внешнего состояния при каждом вызове. (Трейд-офф: жертвуем CPU ради RAM).
* Сильно усложняет код программы.

## 🔍 Как отличить от других паттернов
* **Flyweight vs Singleton:** Singleton гарантирует, что в системе есть ТОЛЬКО ОДИН объект класса. FlyweightFactory может хранить *множество* объектов (Пул), каждый из которых описывает свое внутреннее состояние. Кроме того, объекты Flyweight неизменяемы (immutable).
