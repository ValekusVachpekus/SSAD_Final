# Prototype (Прототип)
**Категория:** Creational (Порождающий)

## 📌 Суть (Intent)
Позволяет **копировать (клонировать)** существующие объекты, не привязывая клиентский код к их конкретным классам.

## 🛠 Проблема и Решение
**Проблема:** Вам нужно создать точную копию объекта. Если делать это снаружи (через `new` и копирование полей), клиентский код должен знать точный класс объекта, а некоторые поля могут быть `private` и недоступны.
**Решение:** Делегировать процесс клонирования самому объекту через общий интерфейс с методом `clone()`.

### Пример кода (Лекция: Фигуры / Лаба 9: Клонирование Car / Pandora: Клонирование Creature)
```cpp
class Car { // Интерфейс прототипа
public:
    virtual Car* clone() const = 0;
    virtual void specs() const = 0;
    virtual ~Car() {}
};

class Sedan : public Car {
private:
    std::string color;
public:
    Sedan(const std::string& color) : color(color) {}
    
    // Ключевой метод! Возвращает новую копию самого себя
    Car* clone() const override {
        return new Sedan(*this); // Вызов конструктора копирования
    }
    
    void specs() const override {
        std::cout << "Sedan Car - Color: " << color << std::endl;
    }
};

// В main():
Car* originalCar = new Sedan("Red");
Car* clonedCar = originalCar->clone(); // Клонируем, не зная точного типа!
```

## ⚖️ Плюсы и Минусы
**Плюсы:**
* Клонирование без жесткой привязки (Coupling) к конкретным классам.
* Позволяет избавиться от тяжелого кода инициализации (просто клонируем готовый "пресет").
* Удобно хранить реестр прототипов (Prototype Registry).

**Минусы:**
* Клонирование сложных объектов, содержащих **циклические ссылки** (circular references), может быть настоящей болью.

## 🔍 Как отличить от других паттернов
* **Prototype vs Builder:** Prototype клонирует уже *готовый* и настроенный объект. Builder конструирует объект *шаг за шагом* с нуля.
