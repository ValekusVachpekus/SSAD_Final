# Memento (Снимок / Хранитель)
**Категория:** Behavioral (Поведенческий)

## 📌 Суть (Intent)
Паттерн **Memento** позволяет сохранять и восстанавливать прошлые состояния объекта (делать Undo), **не раскрывая детали его реализации (не нарушая инкапсуляцию)**.

## 🧠 Фундаментальная идея (из Лекции 15)
**Проблема (Пример: Текстовый редактор):** Мы хотим сделать кнопку Undo (Отмена). Чтобы отменить изменение текста, нужно сохранить бэкап объекта `Editor`. Если класс `Editor` откроет все свои поля (`string text`, `int cursorPosition`) как `public`, любой сможет их поменять, нарушится инкапсуляция. Если оставит `private` — внешний класс `History` не сможет их прочитать, чтобы сохранить!
**Решение:** Пусть сам `Editor` (Создатель/Originator) создает снимок (Memento) своего состояния. Так как снимок создается *внутри* Editor'а, он имеет доступ ко всем приватным полям! Затем этот снимок передается на хранение внешнему классу (Caretaker/Опекун). Опекун **не может** прочитать или изменить снимок (поля внутри снимка закрыты), он может только хранить его.

## 🛠 Архитектура (Роли)
1. **Originator (Создатель):** Объект, чье состояние мы сохраняем (`Editor`). Имеет методы `save()` (возвращает Memento) и `restore(Memento)`.
2. **Memento (Снимок):** Объект-значение (Value Object), хранящий состояние Создателя. Поля снимка строго `private` или `const`, изменять его после создания нельзя. У него нет публичных геттеров/сеттеров.
3. **Caretaker (Опекун):** Объект, который знает "когда" и "почему" нужно сохранять состояние (например, `UndoRedoManager` или `CommandHistory`). Хранит стек Снимков, но **не имеет доступа** к их содержимому.

## 💻 Пошаговая реализация и Примеры

### Пример: Редактор текста с Undo/Redo (Лекция 15 / Лаба 15)
Чтобы `Caretaker` не мог лезть в `Memento`, в C++ часто используют вложенные классы (Nested Classes) или `friend class`.

    // 1. Снимок (Все данные спрятаны!)
    class EditorMemento {
    private:
        friend class Editor; // ТОЛЬКО Editor имеет доступ внутрь снимка!
        const string text; // Состояние неизменяемо (immutable)
        
        EditorMemento(string t) : text(t) {} // Конструктор скрыт от чужих
        string getText() { return text; }
    };

    // 2. Создатель (Тот, чье состояние сохраняем)
    class Editor {
    private:
        string content = "";
    public:
        void type(string words) { content += words; }
        
        // Создатель делает снимок САМ
        EditorMemento* save() {
            return new EditorMemento(content);
        }
        
        // Восстановление из снимка
        void restore(EditorMemento* memento) {
            content = memento->getText(); // Имеет право, так как он friend
        }
        string getContent() { return content; }
    };

    // 3. Опекун (Хранит снимки, но ничего с ними сделать не может)
    class History {
        vector<EditorMemento*> history;
    public:
        void backup(EditorMemento* m) { history.push_back(m); }
        
        EditorMemento* undo() {
            if (history.empty()) return nullptr;
            EditorMemento* m = history.back();
            history.pop_back();
            return m;
        }
    };

    // Клиент:
    Editor editor;
    History hist;
    
    editor.type("Hello ");
    hist.backup(editor.save()); // Сохранили
    
    editor.type("World!");
    cout << editor.getContent(); // "Hello World!"
    
    editor.restore(hist.undo()); // Откатили!
    cout << editor.getContent(); // "Hello "

## ⚖️ Плюсы и Минусы
**Плюсы:**
* Не нарушает инкапсуляцию (приватные поля объекта остаются приватными, никто снаружи их не читает).
* Упрощает класс Создателя (ему не нужно самому хранить массивы своих прошлых состояний, это делает Caretaker).

**Минусы:**
* **Жрет оперативную память (RAM).** Если состояние объекта огромное, а снимки делаются часто, память улетит в небеса. (Решение: сохранять не весь объект, а только "дельту" изменений).

## 🔍 Как отличить от других паттернов
* **Memento vs Command:** Очень часто работают вместе. Команда (`Command`) отвечает за выполнение действия и содержит логику `undo()`. А внутри себя Команда хранит Снимок (`Memento`), чтобы знать, к какому состоянию возвращать объект при отмене.
