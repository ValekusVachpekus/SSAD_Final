# Singleton (Одиночка)
**Категория:** Creational (Порождающий)

## 📌 Суть (Intent)
Гарантирует, что у класса есть **только один экземпляр**, и предоставляет к нему **глобальную точку доступа**.

## 🛠 Реализация (Как это сделать)
1. Скрыть конструктор (сделать его `private`), чтобы никто не мог вызвать `new Singleton()`.
2. Создать `private static` поле, которое будет хранить единственный экземпляр.
3. Создать `public static` метод `getInstance()`, который:
   * Проверяет, создан ли объект.
   * Если нет (Lazy initialization) — создает его.
   * Возвращает существующий инстанс.

### Пример кода (на основе Лекции/Лабы 9 - Logger)
```cpp
class Logger {
private:
    static Logger* instance;
    // 1. Приватный конструктор
    Logger() {} 

public:
    // Запрещаем копирование и присваивание! (Важно для C++)
    Logger(const Logger&) = delete;
    Logger& operator=(const Logger&) = delete;

    // 2. Глобальная точка доступа
    static Logger* getInstance() {
        if (instance == nullptr) {
            instance = new Logger();
        }
        return instance;
    }

    void log(const std::string& message) {
        std::cout << "[LOG]: " << message << std::endl;
    }
};
// Инициализация статического поля вне класса
Logger* Logger::instance = nullptr;
```

## ⚖️ Плюсы и Минусы
**Плюсы:**
* Уверенность, что объект только один (полезно для БД, логгеров, кэша).
* Глобальная точка доступа.
* Отложенная (Lazy) инициализация (создается только при первом запросе).

**Минусы:**
* Нарушает **Single Responsibility Principle (SRP)** (класс контролирует и свою логику, и свое создание).
* Маскирует плохой дизайн (глобальное состояние).
* **Сложности в многопоточности** (Thread safety). В Java нужен `volatile` и `synchronized` (double-checked locking). В C++11 статические локальные переменные потокобезопасны.
* Усложняет Unit-тестирование (нельзя просто так замокать).

## 🔍 Как отличить от других паттернов
* **Singleton vs Статический класс:** Статический класс нельзя передать как параметр или реализовать им интерфейс. Singleton — это полноценный объект.
* Единственный паттерн, который искусственно **ограничивает** создание своих объектов на уровне языка (`private` конструктор).
